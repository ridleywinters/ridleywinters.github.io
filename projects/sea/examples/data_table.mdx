
# Data Table

Sometimes it may be helpful to have a one-off presentation of data for a specific page, where creating a global `Renderer` and broader in scope than needed.

Here's an example. This is a list of tools/technologies/products that we want to create Google search links for each. Rather than typing out each, let's create a simple list of product names and some inline JSX to display that inline list.

### Results

```@data name=list, pizza=salad
- Backblaze
- Google One
- Visual Studio Code
- Blender
- GIMP
- Chrome
```

<div>
    {_.map(page.name.list.value, (item, i) => (
        <div key={i} style={{ display : 'flex', flexDirection: 'row', alignItems: 'center' }}>
            <div style={{ fontSize: '24px', paddingRight : '12px' }}>• </div>
            <div>
                <a href={`http://google.com/search?q=${encodeURIComponent(item)}`}>{item}</a>
            </div>
        </div>
    ))}
</div>

### Concepts

* `@data` objects
* Inline JSX
